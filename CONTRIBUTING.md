# Contributing to Train or Be Trained

Thanks for your interest in improving the project.

## Best ways to contribute

The highest-value contributions right now are content contributions that improve the quality, clarity, and usefulness of generated Working With Me output.

Examples:
- clearer instruction wording
- contradiction cleanup between dimensions
- sharper challenge/rigor balance
- concise docs improvements for real-world usage

## Local development

- Install dependencies: `npm install`
- Start local app: `npm start`
- Run tests: `npm test`
- Build for production: `npm run build`

## Pull request expectations

- Keep PRs focused and small.
- Explain the user-facing impact in plain language.
- Include tests or evidence for behavioral changes.
- Update docs when behavior or workflow changes.

Large product-shape changes may be declined during beta while the current model is being validated.

## Current beta boundaries

- No backend or account features in this phase.
- No new controls/dimensions in this phase.
- Prioritize stability, coherence, and practical collaboration outcomes.

## Where to start

Please use GitHub Issues as the primary path for bug reports, proposals, and contribution discussion.
