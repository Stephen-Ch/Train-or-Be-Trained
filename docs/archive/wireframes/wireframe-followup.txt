Behavior & Rules
1) Exactly ONE follow-up card visible at a time.
2) Continue advances index; Skip records skipped and advances.
3) Persistence: answers save immediately. Refresh resumes current FU.
4) Breadcrumb uses category label and a shortened TLQ label (first 6–8 words).
5) A11y: fieldset/legend for radio groups; focus management moves to next card on advance.
6) Navigation outcome:
   - if more FUs for current TLQ → next FU;
   - elif more TLQs in category → first FU of next TLQ;
   - else → return to /q/:id top-level screen (all TLQs answered).

Selectors (stable)
- Breadcrumb:       data-testid="breadcrumb"
- FU progress:      data-testid="followup-progress"
- FU card:          data-testid="fu-card-<tlqId>-<k>"
- FU statement:     data-testid="fu-statement-<tlqId>-<k>"
- Likert radios:    data-testid="likert-<tlqId>-<k>-value-1..5"
- Buttons:          data-testid="skip", data-testid="continue"
