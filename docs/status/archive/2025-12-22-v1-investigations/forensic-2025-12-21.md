## Bug Analysis Performed

### Bug #1: Category Order Mismatch

**Root Cause Identified:**
```typescript
// session.store.ts line 49
private _sequence = computed(() => [...this._selectedIds()].sort());
```

The `.sort()` call alphabetized categories:
- User selects: `["liberty", "equality"]`
- Sequence becomes: `["equality", "liberty"]` (alphabetical)
- `"equality" < "liberty"` alphabetically confirmed via PowerShell

**Verification:**
```powershell
PS> "equality" -lt "liberty"
True
```

### Bug #2: TLQ/Followup Answer Key Collision (TD-RAWLS-003)

**Root Cause (Previously Identified):**
Content model uses same `id` for TLQ and followups (e.g., `liberty-q0`).

When user answers TLQ with value 4:
```json
{ "liberty-q0": 4 }
```

Followups phase reads same key, causing pre-selected radios.

**Fix Already Deployed (Commit 2cfc440):**
```typescript
private followUpAnswerKey(categoryId: string, tlqId: string, followupIndex: number): string {
  return `fu:${categoryId}:${tlqId}:${followupIndex}`;
}
```

### Bug #3 & #4: Continue Button + Signal Timing
Same root cause as Bug #2 — if old code running, collision persists.

## Fix Implementation: Bug #1 (Category Order)

### Code Changes

**File: session.store.ts**

1. Added imports:
```typescript
import { Injectable, signal, computed, inject } from '@angular/core';
import { ContentService } from '../content/content.service';
```

2. Injected ContentService:
```typescript
export class SessionStore {
  private contentService = inject(ContentService);
```

3. Changed sequence computation:
```typescript
// OLD: Alphabetical
private _sequence = computed(() => [...this._selectedIds()].sort());

// NEW: Content-file order
private _sequence = computed(() => {
  const selected = this._selectedIds();
  const categories = this.contentService.state().categories;
  
  // If content not loaded yet, preserve insertion order
  if (categories.length === 0) {
    return [...selected];
  }
  
  // Sort by position in content file
  const orderMap = new Map(categories.map((c, i) => [c.id, i]));
  return [...selected].sort((a, b) => {
    const posA = orderMap.get(a) ?? Infinity;
    const posB = orderMap.get(b) ?? Infinity;
    return posA - posB;
  });
});
```

## Key Learnings

1. **Alphabetical sorting was silent bug** — Tests passed because they expected alphabetical order, matching the implementation. The bug was in the design decision, not the code.

2. **Content-file order requires ContentService injection** — SessionStore now depends on ContentService, creating a coupling. This is acceptable because sequence order is a presentation concern tied to content.

3. **Caching issues remain unresolved** — Multiple attempts to clear cache failed. Root cause may be:
   - Multiple node processes
   - Service worker caching
   - Angular dev server aggressive caching
   - Stale dist folder

4. **Test mocks must match production behavior** — When changing sequence logic, all tests using sequence needed mock ContentService updates.
