### 4. FW-ADMIN-001: Admin Content Editor MVP (Read-Only + Safe Text Edits)

**Description**: Grouped UI for viewing and editing existing text fields only (category name/desc, TLQ prompt, follow-up prompt). **No add/delete/reorder in V1.1.**

**Acceptance Criteria**:
- Admin navigates to /admin/content → All categories render in grouped tree view with expand/collapse and search/filter
- Edit existing category name/description in-place → Save updates content JSON and passes validation
- Edit existing TLQ/follow-up prompt text → Save updates content JSON and passes validation
- Save errors surfaced clearly in UI (no silent failures)
- Save runs through content-build.js pipeline → validation errors block save
- V1.1 excludes: add new items, delete items, reorder/drag-drop, ID generation

**Test Plan**:
- Contract test: edits preserve schema structure
- Integration test: saved content.json passes content.integrity.spec.ts

**Risk**: Schema validation regression if save logic bypasses pipeline

## Why This Ordering

Tests first (item 1) to catch regression risks early. Low-risk copy changes next (item 2) for fast UX value with zero schema risk. UI polish (item 3) adds user-facing quality without content migration. Admin MVP (item 4) delivers editor capability with tightly scoped safe edits only. Docs versioning (item 5) supports long-term protocol hygiene with minimal urgency.

---

## Commit Plan

Each item committed separately with test gates verified before merge. Sequence follows sprint slice order: item 1 (real-content test) → item 2 (terminology rename) → item 3 (UI polish) → item 4 (admin MVP) → item 5 (docs versioning).
